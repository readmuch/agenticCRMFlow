## 1. agenticCRM 소개

**agenticCRM**은 기관투자자 대상 영업 담당자를 위한 AI 기반 CRM입니다. Claude 계열 AI 에이전트가 **고객 데이터와 영업 노트를 읽어 자동으로 네 가지 산출물을 생성**합니다.

| 산출물 | 무엇을 해주나 | 어디에 쓰나 |
|---|---|---|
| **Persona** (페르소나) | 고객의 선호 섹터·콘텐츠 유형·분석 스타일·**싫어하는 것**을 정리 | 다음 미팅·자료 준비 시 "이 고객이 좋아할 방식" 참고 |
| **NBA** (Next Best Action) | 우선순위별로 다음에 해야 할 영업 액션 추천 | 이번 주·이번 달 활동 계획 수립 |
| **Activity** | NBA를 날짜가 붙은 구체적 **실행 일정**으로 변환 | 캘린더·할 일 리스트로 활용 |
| **QC 보고서** | 위 세 산출물의 **품질 점수**와 일관성 검수 | 결과를 신뢰해도 되는지 빠르게 판단 |

### 어떤 점이 다른가

- **영업 노트가 핵심 입력**입니다. 고객을 만난 후 적은 노트의 `Customer_Feedback`과 `Action_Point` 두 필드가 모든 AI 분석의 근거가 됩니다.
- **증분 분석**이 기본이라 이미 분석된 고객이라도 **새 노트만 읽어서 빠르게** 업데이트됩니다.
- **전체 고객 일괄 처리**가 가능해 매주 한 번 버튼 한 번으로 11명 전 고객 재분석이 가능합니다.

## 5. 대시보드 — 일상 업무의 출발점

### 5.1 화면 구성

![대시보드 일괄 분석 버튼](./images/02_dashboard_bulk_buttons.png)

대시보드 상단에는 다음이 있습니다.

- **통계 카드 3개**: 전체 고객 수 / 분석 완료 수 / 미분석 수
- **일괄 실행 버튼 4개** (🆕):
  - `[전체 페르소나 업데이트]` (파랑) · 옆에 `강제 재생성` 체크박스
  - `[전체 NBA 추천]` (노랑) · 옆에 `강제 재생성` 체크박스
  - `[전체 Activity 업데이트]` (초록)
  - `[전체 QC 검수]` (하늘)
- **뷰 토글**: `[카드]` / `[리스트]`

### 5.2 고객 카드 읽는 법

각 카드에는 아래 정보가 표시됩니다.

| 영역 | 내용 |
|---|---|
| 상단 | 회사명 · 등급 배지 (S/A/B) |
| 중단 | 고객 유형(자산운용사·헤지펀드 등) · 운용자산(AUM) |
| 담당자 | 담당 컨택 이름 · 직함 · 이메일 |
| 하단 태그 | 관심 섹터·투자 스타일 태그 |
| 최하단 | `✅ 분석 완료` 배지 + **노트 N건 · 최근 활동일** (🆕 2026-04-17) |

**카드 전체가 클릭 가능**합니다. 클릭하면 해당 고객의 상세 분석 페이지(`/customer/{id}`)로 이동합니다.

### 5.3 카드 뷰 ↔ 리스트 뷰

우측 상단의 `[카드] / [리스트]` 토글로 전환할 수 있습니다. 리스트 뷰는 데이터가 많을 때 유용합니다.

> 💡 **팁**: 탭을 바꿔도 마지막으로 본 탭은 `localStorage`에 저장되어 재방문 시 자동 복원됩니다.

## 6. 🆕 AI 챗 사이드바 — 데이터에 말로 질문하기

**추가일**: 2026-04-24

![AI 챗 사이드바](./images/03_ai_chat_sidebar.png)

### 6.1 열고 닫기

- 화면 **우측 하단의 파란색 챗 버튼**(🤖 아이콘)을 클릭하면 사이드바가 열립니다.
- 우측 상단의 `X` 버튼으로 닫고, ↻ 버튼으로 **대화 초기화**가 가능합니다.

### 6.2 사용법

입력창에 자연어 질문을 입력하고 `Enter`를 누릅니다. (줄바꿈은 `Shift+Enter`)

**질문 예시**:

- "미래에셋자산운용의 최근 페르소나 요약해줘"
- "이번 주 기한이 임박한 Activity 어떤 게 있어?"
- "불만 징후가 감지된 고객이 누구야?"
- "QC 점수가 80점 미만인 고객 알려줘"
- "반도체 섹터에 관심 있는 고객 리스트 보여줘"

### 6.3 동작 원리

AI 챗(`ChatAgent`)은 **읽기 전용 도구**만 사용하여 다음 데이터에 접근합니다.

- 전체 고객 목록 · 특정 고객 상세
- 페르소나 · NBA · Activity · QC 결과
- 영업 노트

> ⚠️ **주의**: 챗 에이전트는 **수정·삭제·생성**을 수행할 수 없습니다. 오직 조회·요약·비교만 합니다.

> 📝 **대화 이력**: 서버는 대화를 저장하지 않습니다. 브라우저 탭을 닫거나 새로고침하면 이력이 사라집니다.

### 6.4 응답 모델

챗 응답은 **화면 상단의 모델 셀렉터**에서 선택한 모델이 사용됩니다. [§13](#13-분석-모델-변경하기) 참고.

## 7. 🆕 전체 고객 일괄 분석 (Bulk Pipeline)

**추가일**: 2026-04-23 ~ 2026-04-24

전체 11명의 고객을 한 번에 AI 분석하는 기능입니다. 매주 월요일 아침 루틴으로 쓰기에 최적입니다.

### 7.1 네 가지 일괄 실행 버튼

대시보드 상단에서 원하는 단계의 버튼을 누릅니다.

| 버튼 | 수행 내용 | 의존성 | 강제 재생성 체크박스 |
|---|---|---|---|
| 🔵 **전체 페르소나 업데이트** | 각 고객의 최신 `Customer_Feedback` 노트를 읽어 페르소나 생성·갱신 | 없음 | ✅ 있음 |
| 🟡 **전체 NBA 추천** | 페르소나 + 최근 3개월 노트로 NBA 액션 추천 | Persona 필요 (없으면 스킵) | ✅ 있음 |
| 🟢 **전체 Activity 업데이트** | NBA를 날짜가 붙은 실행 일정으로 변환 | NBA 필요 (없으면 스킵) | ❌ |
| 🔷 **전체 QC 검수** | 페르소나/NBA/Activity 출력을 종합 검수 | Persona · NBA · Activity 모두 필요 | ❌ |

### 7.2 실행 흐름

1. 버튼을 클릭하면 바로 아래 **진행 패널**이 펼쳐집니다.
2. 진행 패널에는 다음이 실시간 표시됩니다.
   - 상단: `N / 11` 카운터 배지
   - 중간: 진행률 바 (줄무늬 애니메이션)
   - 하단: 고객별 상태 로그
     - `분석 중…` (파랑)
     - `✅ 완료`
     - `⏭️ 스킵` (사유 명시, 예: "페르소나 없음")
     - `⚠️ 오류` (에러 메시지)
3. 완료 시 요약: **성공 N · 실패 N · 스킵 N · 완료 시각 (KST)**

![일괄 분석 진행 패널](./images/04_bulk_progress_panel.png)

### 7.3 "강제 재생성" 토글 (🆕 2026-04-21)

페르소나·NBA 버튼 옆의 체크박스입니다.

- **체크 해제 (기본)**: **증분 모드**. 마지막 분석 이후 **추가된 노트만** 반영. 빠르고 비용이 적습니다.
- **체크 ✅**: **전체 재생성**. 모든 노트를 처음부터 다시 읽어 분석. 고객 선호가 크게 바뀌었거나, 이전 분석을 통째로 덮어쓰고 싶을 때 사용.

> 💡 **권장**: 평소엔 끄고, 분기별 1회 또는 중요 변곡점(상품 전략 변경 등)에만 켭니다.

### 7.4 의존성 자동 스킵

잘못된 순서로 실행해도 안전합니다.

- NBA 버튼 → 페르소나 없는 고객은 자동 `스킵`
- Activity 버튼 → NBA 없는 고객은 자동 `스킵`
- QC 버튼 → Persona/NBA/Activity 중 하나라도 없으면 자동 `스킵`

> 💡 **권장 순서**: 페르소나 → NBA → Activity → QC. 한 단계씩 확인하며 진행하는 걸 권장합니다.

## 8. 개별 고객 AI 분석 — Persona · NBA · Activity · QC

대시보드에서 고객 카드를 클릭하면 **고객 상세 페이지**(`/customer/{id}`)로 이동합니다.

![고객 상세 페이지](./images/05_customer_detail_overview.png)

### 8.1 상단 헤더

- 회사명 · 등급 · 유형 · AUM · 담당자 정보
- **모델 셀렉터** (🆕 2026-04-23): 이 고객 분석에 사용할 AI 모델을 여기서도 변경 가능

### 8.2 "전체 분석 실행" 카드

`[전체 분석 시작]` 버튼을 누르면 **Persona → NBA → Activity → QC 순차 파이프라인**이 실행됩니다.

- 실시간 로그 터미널이 펼쳐져 각 에이전트의 진행 상황이 스트리밍됩니다.
- 중간 단계가 실패하면 자동 재시도합니다 (QC가 `fail` 판정 시 최대 1회).

### 8.3 네 가지 섹션 카드

아래로 스크롤하면 네 섹션이 카드 형태로 있습니다.

#### 8.3.1 페르소나 분석

- 회사명이 헤더에 표시됩니다 (🆕 2026-04-17).
- 고객 선호도 프로파일:
  - 선호 섹터
  - 선호 콘텐츠 유형
  - 분석 스타일 (예: 숫자 중심, 스토리 중심)
  - 명시적 요구사항
  - **explicit_dislikes** — 고객이 **명시적으로 거부·불만을 표현한 항목** (§10 Red-flag 판정의 근거)
- 상단 `[Persona 업데이트]` 버튼으로 단독 재분석
- 🆕 우측에 **마지막 업데이트 시각** (KST, 분 단위)

#### 8.3.2 NBA 추천

- 우선순위별 영업 액션 목록
- 각 액션에 **구체적 기한 (YYYY-MM-DD)**
- **참고 노트 비교 테이블**: AI가 이 NBA를 추천한 근거가 된 과거 노트 행이 연결 표시됩니다.
- **최우선 액션 vs 노트 Action_Point 비교 패널** (🆕 2026-04-23)
- 🆕 매칭 노트가 **Red-flag** 로 플래그된 경우 **빨간 경고 배너 + 카드 테두리** 강조
- `[NBA 제안]` 버튼으로 단독 재분석

#### 8.3.3 Activity 일정

- NBA 액션을 날짜 기반 일정으로 변환한 표
- **진행 상태** 컬럼: `pending` / `in_progress` / `completed` / `cancelled`
- **NBA 승인 상태** 미러링 컬럼 (§14 참고)
- `[Activity 업데이트]` 버튼으로 단독 재분석

#### 8.3.4 QC 검수

- **큰 원형 점수** (0~100)
- **판정 배지**: `pass_excellent` / `pass_good` / `conditional_pass` / `fail`
- 부문별 점수 4개: 페르소나 / NBA / Activity / 일관성
- **Critical Issues**: 심각도(`critical`/`major`/`minor`)별 색상 표시 + 각 이슈의 `fix` 제안
- `[QC 검수 실행]` 버튼으로 단독 재분석

### 8.4 영업 노트 아코디언

섹션 위쪽에는 **과거 활동 기록 아코디언**이 있습니다. 각 항목은 노트 1건이며, 펼치면 `Activity_Log`·`Customer_Feedback`·`Action_Point`가 모두 보입니다. `Action_Point`는 굵게 강조 표시됩니다.

**새 노트 추가**: 아코디언 상단의 `[노트 추가]` 버튼으로 모달이 열립니다. (§9 참고)

## 10. 🆕 고객 불만 징후(Red-flag) 탐지

**추가일**: 2026-04-23

영업 담당자가 적은 `Action_Point`가 해당 고객이 **이전에 명시적으로 거부·불만을 표현한 항목**(페르소나의 `explicit_dislikes`)과 실제로 충돌하는지 AI가 자동 판정하는 기능입니다.

### 10.1 왜 중요한가

예시 시나리오:

- **고객 페르소나**: "원자재 관련 분석은 받지 않겠다고 명시"
- **새 노트 Action_Point**: "원자재 섹터 리포트 발송 예정"
- → **Red-flag 🚩** 자동 탐지

이런 충돌이 실제 고객에게 전달되기 전에 영업 담당자에게 경고를 띄워 신뢰 하락을 막습니다.

### 10.2 실행 방법

1. **탭**: `세일즈 노트 전체 조회`
2. 판정하고 싶은 노트들의 체크박스를 선택
3. `[고객 불만 징후 탐지]` 버튼 (노랑) 클릭
4. `DislikeCheckerAgent`가 고객별로 배치 처리 → 결과 저장 → 테이블 재렌더링

### 10.3 결과 표시

#### 세일즈 노트 테이블
- Red-flag 행은 **빨간 배경**으로 강조
- `고객불만징후감지` 컬럼에 `🚩 YES` 뱃지
- `감지 이유` 컬럼에 판정 근거 한 문장

#### 고객 상세 페이지 NBA 섹션
- 해당 노트가 NBA 비교 패널의 `top_priority_comparison`에 연결되어 있으면 **빨간 경고 배너 + 빨간 카드 테두리** 자동 표시

#### 전체 NBA 탭
- 동일한 경고 스타일이 전 고객 비교 화면에도 적용됩니다 (추가 AI 호출 없이 메타 조인).

### 10.4 페르소나가 없는 고객은?

`explicit_dislikes`가 비어 있거나 페르소나가 없는 고객은 자동 스킵됩니다. 먼저 페르소나 분석을 실행하세요.

| 대량 일괄 업데이트 | **Claude Sonnet 4.6** |
| 테스트·데모·예산 절약 | **OpenRouter 무료 모델** |

> ⚠️ **무료 모델 주의**: rate limit으로 실패할 수 있습니다. 실패 시 다른 모델로 재시도하거나 시간 간격을 두세요. 무료 모델은 때로 JSON 형식을 정확히 맞추지 못해 일부 필드가 비어 있을 수 있습니다 (시스템이 자동 backfill).

### 13.3 모델 변경 후 적용 시점

모델을 변경하면 **다음에 실행하는 모든 에이전트**가 해당 모델을 사용합니다. 진행 중인 실행에는 영향을 주지 않습니다.

## 14. NBA 승인 워크플로우

NBA 추천 결과의 각 액션은 **3단계 승인 상태**를 가집니다.

| 상태 | 설명 | 배지 색상 |
|---|---|---|
| `ai_proposed` | AI가 NBA를 제안한 초기 상태 | 🟡 노랑 |
| `crm_approved` | CRM 담당자가 검토·승인한 상태 | 🔵 파랑 |
| `sales_approved` | 세일즈 담당자가 최종 승인, 즉시 실행 가능 | 🟢 초록 |

### 승인 단계 의미

- **ai_proposed** — 참고용. 실제로 실행하기 전에 사람의 검토가 필요합니다.
- **crm_approved** — CRM 기획팀·데스크 검토 완료. 예산·리스크·규정 관점에서 OK.
- **sales_approved** — 최종 영업 담당자가 고객에게 전달·실행해도 된다고 판단한 상태.

### Activity와의 연결

Activity 섹션의 각 일정은 연결된 NBA 액션의 승인 상태를 **자동으로 미러링**합니다. 별도로 관리할 필요 없이, NBA 승인 상태를 바꾸면 Activity 화면에도 즉시 반영됩니다.

단, **진행 상태 (pending/in_progress/completed/cancelled)**는 Activity별로 따로 관리됩니다.

## 15. 자주 묻는 질문 (FAQ)

### Q1. 분석이 "증분 모드"로 돌면 어떤 기준으로 새 노트를 인식하나요?

각 에이전트는 마지막 실행 타임스탬프(`updated_at`)를 기록하며, 그 이후 `Activity_Date`가 더 최근인 노트만 분석 대상에 포함됩니다. 전체 재분석을 원하면 **강제 재생성** 체크박스를 사용하세요.

### Q2. 노트를 삭제하면 이전 페르소나·NBA도 사라지나요?

아니요. 노트 삭제는 **분석 결과를 유지**합니다. 페르소나·NBA를 재생성하고 싶다면 해당 에이전트를 다시 실행하세요.

### Q3. AI 챗의 대화 이력은 저장되나요?

아니요. `ChatAgent`는 무상태(stateless)입니다. 브라우저 탭을 닫거나 새로고침하면 대화가 사라집니다. 중요한 답변은 따로 저장해 두세요.

### Q4. Red-flag 탐지를 어떤 주기로 돌려야 하나요?

새 노트를 업로드한 후, 그리고 페르소나가 업데이트된 후에 돌리는 것을 권장합니다. `Action_Point`가 업데이트된 후 1회 돌리면 충분합니다.

### Q5. 일괄 분석 도중 브라우저를 닫으면 어떻게 되나요?

서버측 처리는 **계속 진행**됩니다. SSE 연결만 끊어지므로 실시간 진행 표시만 사라집니다. 브라우저를 다시 열고 대시보드로 돌아오면 최신 상태가 반영되어 있습니다.

### Q6. 모든 시간 표시가 KST인가요?

네. 🆕 2026-04-23부터 모든 사용자 노출 시각은 한국 표준시(UTC+9), 분 단위로 통일되어 있습니다.

### Q7. 한 고객의 노트가 다른 고객 분석에 섞이지 않나요?

🆕 2026-04-17에 cross-customer contamination 버그가 수정되었습니다. 각 에이전트는 해당 `customer_id`의 노트만 참조합니다.

## 16. 문제 해결 (Troubleshooting)

### 에이전트 실행이 계속 실패합니다

1. **화면 상단의 모델**을 확인하세요. OpenRouter 무료 모델은 rate limit 제약이 잦습니다 → Claude Sonnet/Opus로 변경
2. 해당 고객에 **페르소나가 있는지** 확인. NBA·Activity·QC는 상위 산출물을 필요로 합니다.
3. 영업 노트에 `Customer_Feedback` 내용이 **비어 있지 않은지** 확인. 비어 있으면 페르소나 분석 결과가 약합니다.

### 무료 모델 결과에 필드가 누락됩니다

🆕 시스템이 자동으로 `top_priority_comparison` 등 핵심 필드를 backfill하지만, 완벽하지 않을 수 있습니다. 중요한 분석은 Claude 모델을 사용하세요.

### "분석 중…"에서 멈춰 있어요

실제로는 서버에서 진행 중이지만 SSE 연결이 끊긴 경우가 많습니다. 페이지를 새로고침하면 최신 결과가 반영됩니다.

### CSV 업로드 시 모든 행이 "오류"로 표시됩니다

다음을 확인하세요.

- 필수 컬럼이 모두 포함되어 있는지: `Activity_Date`, `Customer_Feedback`, `Action_Point`
- `Activity_Date`가 `YYYY-MM-DD` 형식인지
- `Client_Name`이 기존 고객 목록과 **정확히 일치**하는지 (없으면 행이 스킵됩니다)

## 17. 용어집

| 용어 | 의미 |
|---|---|
| **Persona (페르소나)** | 고객의 선호·스타일·싫어하는 것을 AI가 정리한 프로파일 |
| **NBA (Next Best Action)** | "다음에 해야 할 최적의 영업 액션" — 우선순위 + 기한 + 근거 노트 |
| **Activity** | NBA를 구체적 날짜·제목·유형으로 바꾼 실행 일정 |
| **QC (Quality Control)** | 페르소나·NBA·Activity 출력의 품질을 점수·판정으로 검수하는 단계 |
| **Red-flag** | 영업 노트의 `Action_Point`가 고객의 `explicit_dislikes`와 충돌한다는 경고 |
| **증분 모드** | 마지막 분석 이후 **추가된 노트만** 반영하는 기본 실행 방식 |
| **강제 재생성** | 모든 노트를 처음부터 다시 읽는 전체 재분석 방식 |
| **explicit_dislikes** | 고객이 명시적으로 거부·불만을 표현한 항목 (페르소나의 핵심 필드) |
| **Action_Point** | 영업 노트에서 "후속 조치" 필드 — NBAAgent의 주 참고 + Red-flag 판정 대상 |
| **Customer_Feedback** | 영업 노트에서 "고객 피드백" 필드 — PersonaAgent의 **유일한** 분석 소스 |
| **SSE** | Server-Sent Events — 일괄 분석의 실시간 진행 표시에 사용되는 서버 스트리밍 |
| **tier / 등급** | S / A / B — 고객의 우선순위 등급 |
| **KST** | 한국 표준시 (UTC+9). 2026-04-23부터 모든 시각 표시의 기준 |
